Class Project Assignment

Overview

Work in teams of 5 to select a dataset and a well-posed question, then apply methods from the course so far to produce a reproducible, well-evaluated analysis.

Potential sources for data and ideas include Kaggle, Huggingface, and Havard's Dataverse.

Learning Goals

  • Frame a researchable question from real data.
  • Build a reproducible Python pipeline.
  • Fit and compare models.
  • Evaluate on held-out data.
  • Communicate findings clearly.

Proposal (max 1 page, PDF) — due in 2 weeks.

  • Dataset (name + link/source) and question.
  • Task type (regression, classification, ranking, clustering).
  • Modeling plan, including a simple baseline to be used as a comparison.

Project - due the evening before the last class meeting.

  • Code (GitHub repo)

    • Contains a single .py script that runs the entire analysis end-to-end.

  • Report (≤4 pages, 1.5 spacing, not including figures and tables or references)

  • Presentation (10 minutes)

    • 5–7 slides: Problem → Data → Method → Results → Takeaways → Limits/Next steps.
    • Every team member should speak.

Reproducible Code Requirements

Here is a suggested project GitHub repo structure.

project/

  • README.md

  • requirements.txt

  • main.py — ENTRY POINT: orchestrates full pipeline (imports modules below)

  • data.py — data loading/cleaning; train/val/test split with fixed seed

  • features.py — preprocessing & feature engineering

  • models.py — baseline + at least one class model; training helpers

  • evaluate.py — held-out evaluation; writes outputs/metrics.json & outputs/predictions.csv

  • data/ — raw data, not needed if big

  • outputs/ — generated results

README.md must include:

  • Project question and motivation
  • Data source & access instructions.
  • Environment setup steps.
  • How to run it, for example:

pip install -r requirements.txt

python main.py --data_path /data --seed 101

  • Expected runtime.

Report Guidelines (≤4 pages, ≤2 figures, 1.5 spacing)

Suggested structure

  1. Question & Motivation — why this matters
  2. Data — source, unit of observation, sample size, key variables, missingness; any down-sampling.
  3. Methods — models, features, preprocessing, hyperparameters, split, metrics; clearly identify the models.
  4. Results — table comparing baseline vs. models on held-out data.
  5. Discussion
  6. References

Presentation (10 minutes)

  • Focus on clarity and key comparisons.
  • Avoid code screenshots.
  • Expect 2–3 questions; every member speaks.

AI use policy

  • Use Generative AI for ideas, for code, for troubleshooting; but you are responsible for the results.

Hints & Good Practices

  • If the dataset is huge, feel free to down-sample.
  • Keep a validation set separate from test; don’t peek.
  • Log metrics and seeds; save intermediate artifacts for speed.

Submission Checklist

  • [ ] Proposal PDF (≤1 page) with data link, baseline, metrics, split plan.
  • [ ] GitHub repo link; main.py runs end-to-end.
  • [ ] requirements.txt
  • [ ] Fixed random seed documented.
  • [ ] outputs/metrics.json and outputs/predictions.csv produced by the script.
  • [ ] Report
  • [ ] Slides for a 10-minute talk.
